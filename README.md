# ShuffleStories 🎲📖

> くだらない物語で、初対面の気まずさをぶっ飛ばせ。

---

## サービス概要

起承転結をユーザーそれぞれが投稿し、シャッフルして物語を生成・共有するアプリです。
初対面のオンライン上での気まずさをなくし、くだらない笑いを通じて自然に打ち解けられる場を作ります。

🔗 URL：（デプロイ後に記載）

---

## このアイデアが生まれたきっかけ

RUNTEQに入学した当初、初めての同期との顔合わせで緊張した経験がありました。
年齢・キャリア・性別がバラバラなメンバーと「何を話せばいいかわからない」「気まずくなったらどうしよう」という不安を感じたことがきっかけです。

「くだらないことで笑い合える環境」があれば、もっと自然に仲良くなれると思い、このアプリを作りました。

---

## 解決する課題

**表面の課題：** 初対面のオンライン上で何を話せばいいかわからない

**本質の課題：** 年齢・キャリア・性別が異なるメンバー同士が共通の話題を見つけられず、気まずい沈黙が生まれてしまう

→ **共通のツールを通じて「くだらない笑い」を共有することで、自然なアイスブレイクを実現します**

### 最初に使ってくれそうな人は？

RUNTEQの同期・チームメンバー。グループワークやミニアプリウィークの顔合わせで自然に使ってもらえることを想定しています。

### なぜ使い続けてもらえるのか？

- URLを共有するだけで全員がすぐ使える（登録不要）
- 他のユーザーの物語も見られるので、眺めるだけでも楽しい
- 毎回シャッフル結果が違うため、何度遊んでも新鮮

## 機能一覧

### MVP（初回リリース）

- [x] 起・承・転・結をそれぞれ投稿できる
- [x] 投稿をシャッフルして物語として表示する
- [x] 他のユーザーの作った物語を一覧で見られる
- [x] 100文字以内のバリデーション（冗長な投稿を防ぐ）
- [x] 禁止用語フィルター

## 懸念点と対策

### 懸念点① シャッフルで意味不明な文章になる

- **問題：** 脈絡のない物語になり、面白さよりも困惑が勝つ可能性がある
- **対策：** 100文字以内のバリデーションで冗長な投稿を防ぎ、短くテンポのよい文章に絞る

### 懸念点② 不適切な投稿が混入する

- **問題：** 禁止用語や不適切な内容が物語に表示される
- **対策：** 禁止用語フィルターのバリデーションを実装する

## 技術スタック

| 項目           | 技術                          |
| -------------- | ----------------------------- |
| フレームワーク | Laravel（PHP）                |
| DB             | PostgreSQL（Heroku Postgres） |
| デプロイ先     | Heroku                        |

### 技術選定の理由

- **Laravel：** PHP・Laravel初学者として、ハンズオンで学習するため。アジャイル型開発を取り入れながら実践的なスキルを習得したい
- **PostgreSQL：** Herokuとの公式連携（Heroku Postgres）があり、7日間という限られた期間でハマりにくい構成として選定
- **Heroku：** 初めてのデプロイ体験として手順が整っており、LaravelとPostgreSQLとの相性も良い
