# ぼくの・わたしのえほん 📖

## アプリURL:https://monogatari-ehon-metappiapp-production.up.railway.app/stories

## 1. サービス概要

僕が、私が主人公。
お子様が主人公（自分）お友達の名前を入力するだけで、
世界にひとつだけのオリジナル絵本が生まれるWebアプリです。

---

### 2. 開発背景

「せんたいヒーローになりたい」「お姫様になりたい」——
子どもたちのシンプルな欲求に、既製品の絵本は応えられていません。

自分が主人公になれる物語が身近にないことへの違和感と、
「あったら子どもはどれだけ喜ぶだろう」という想いが、
このアプリのスタート地点になりました。

---

## 3. 課題の整理

「子どもが主人公に、ヒーロー、ヒロイン」になりたいのに、
それを **手軽に** 実現する手段がない

### 3-2. 本当に解決したい課題は何か

- **なぜ？** → 既製品の絵本は主人公が固定されていて、わが子が主役になれないから
- **なぜ？** → 名前入り絵本サービスは高価・時間がかかる・デジタルに対応していないものが多いから
- **なぜ？** → 親御さんが手軽に「特別な体験」を子どもに与える手段がないから

**→ 本当に解決したい課題：親御さんが手軽に、子どもが主役の特別な物語体験をつくれる場所がない**

---

## 4. 想定ユーザー

**年齢設定：**

- 年齢：4〜8歳の子どもと、その保護者（25〜40代）
- 理由：文字が読めるようになりはじめる年齢で、物語への興味が最も高い時期。
- 保護者が一緒に使うことで親子のコミュニケーションにもなる。

**生活・状況設定：**

- 状況：おやすみ前の読み聞かせの時間、子どもと過ごすちょっとした隙間時間
- 理由：スマートフォン・タブレット1台で完結するため、場所を選ばず使える。
- 物語生成が数十秒で完了するため、短い時間でも十分に楽しめる。

**環境設定：**

- 環境：スマートフォン・タブレット・PC、ブラウザがあれば使える
- 理由：インストール不要・会員登録不要のため、導入ハードルが極めて低い。

### 4-2. 自分とユーザーの距離

**→ 身近な誰か（子どもを持つ親御さん）**

小売業で長年お客様と接してきた経験から、子育て世代のお客様との対話が多くあった。
「子どもと過ごす時間をもっと豊かにしたい」という声を現場で多く聞いてきた原体験がある。

### 4-3. 実際に使われる可能性について

- **最初に使ってくれそうな人：** 子どもがいる知人・友人、RUNTEQ受講生コミュニティ
- **どんな場面で思い出すか：** 寝かしつけのとき、子どもが「おはなしよんで！」と言ったとき
- **置き換えてまで使う理由：** 「わが子が主役」という体験は既製品では安易に代替できない

### 4-4. ユーザーがサービスを導入して利用するまでのイメージ

1. SNSやQRコードでURLを知る
2. 会員登録不要・即座に使える → 離脱ハードルが低い
3. 子どもが喜ぶ → 「また今日もやろう！」と習慣化
4. みんなのものがたり一覧でほかの子の物語をみて楽しむ

## 5. 既存サービス・競合調査

### 5-1. 似たサービスの調査

| サービス名                     | できること                                                       | できないこと                       |
| ------------------------------ | ---------------------------------------------------------------- | ---------------------------------- |
| 名前入り絵本（各種出版社・EC） | 子どもの名前が登場するオリジナル絵本を印刷・製本して届けてくれる | 高価・時間がかかる・デジタル非対応 |
| 絵本ナビ                       | 市販の絵本をデジタルで読める                                     | 既製品のみで子どもが主役になれない |
| PIBO                           | 童話・昔話を中心に360冊以上読める 1日3冊まで無料                 | 既存の物語のみでオリジナル作成不可 |

### 5-2. それでも自分が作りたい理由

どちらのサービスも「既にある物語を読む」という体験に留まっています。
「自分の名前・好きなキャラクターが登場する物語」を読む体験は、
既存のどのサービスでも実現できていませんでした。

### 5-3. 差別化を一文で決める

**「わが子が主役のオリジナル絵本が、デジタルで一瞬で作れる」**
——これが既存サービスにはない、このアプリだけの価値です。

---

## 6. このサービスで提供したい価値

- **行動の変化：** 毎晩の読み聞かせが、既製品の絵本から「わが子オリジナルの物語」に変わる
- **気持ちの変化：** 子どもは「自分が主役！」という特別感を毎日感じられる
- **考え方の変化：** 物語を「受け取るもの」から「一緒につくるもの」として捉えるようになる

---

## 7. このアプリで実現すること

### MVPで作る機能（実装済み）

- ジャンル選択（15種類）
- 主人公・なかま・てき・キーアイテムの入力
- テンプレートを使った物語自動生成
- 絵本風ページめくり表示（表紙・起承転結・裏表紙）
- ジャンルごとの挿絵表示
- みんなのものがたり一覧
- いいね機能
- 利用規約・プライバシーポリシー

### 本リリースで作る機能

- ユーザー登録・ログイン機能（自分の物語を保存・管理）
- 物語のシェア機能（SNS・画像保存）
- テンプレートをジャンルごとに5話以上に増やし、毎回新しい物語を提供
- 管理者画面（テンプレート・ジャンルの追加・編集）

### 収益化について

現時点では未定。

１：現在は結末まで４ページと短めなので、
１０-１５ページ作成などの場合は有料プランとして検討中

２：AI物語作成、画像生成化する場合（予定）同じく有料プランとして検討
(トークン代がかかるため、規模により金額検討)

## 8. このアプリの懸念点とその対策

### 懸念点①：不適切な内容の投稿

- **何が問題になりそうか：** ログイン不要のため、悪意ある内容が投稿される可能性がある
- **なぜそう思うか：** 子ども向けサービスであるため、不適切な投稿が表示されると信頼を失う
- **対策：** バリデーションで文字数制限・使用禁止ワードフィルターを実装予定。将来的にはログイン機能で投稿者を管理する。

### 懸念点②：物語の使い回し感

- **何が問題になりそうか：** テンプレートが少ないと「同じ物語ばかり」になり飽きられる
- **なぜそう思うか：** 現状1ジャンル1テンプレートのため、バリエーションが限られる
- **対策：** 各ジャンル5テンプレート以上に増やす。展開（たのしい・しんけん・おわらい）の選択肢を追加予定。

## 9. 技術スタック

| カテゴリ       | 技術                       | 選定理由                                                                     |
| -------------- | -------------------------- | ---------------------------------------------------------------------------- |
| フレームワーク | Laravel 13.7               | RailsでMVC・ORM・ルーティングの概念を習得済みのため、思想が近いLaravelを選定 |
| 言語           | PHP 8.4                    | Laravel採用に伴い選定。PHP実践力を高めることが目的                           |
| DB             | MySQL                      | Laravel Sailに標準搭載のため採用                                             |
| 開発環境       | Docker（Laravel Sail）     | 環境構築の手間を省き、Laravel・PHPの学習に集中するため                       |

| フォント       | Zen Maru Gothic / Klee One | 絵本らしいやわらかい雰囲気を表現するためGoogle Fontsより選定                 |
